# Requirements Document

## Project Description (Input)
詳細画面のメッセージタイムラインにおいて、ユーザーメッセージの見え方と左右レイアウトのバランスを改善したい。具体的には、ユーザー入力の本体である「My request for Codex」をカード上部に配置し、フォントサイズをアシスタントと揃えて会話として読みやすくする。IDE コンテキスト/「My request for Codex」メニューは不要とし、ユーザーカードは「本文」と任意のオプション項目（デフォルト未選択）のシンプル構成にしたい。また、詳細情報カラムが主表示を圧迫しているため、メインメッセージ領域を約 7 割、右側詳細を約 3 割とするレイアウト比に改めたい。

## Requirements

### Requirement 1: ユーザーメッセージを会話本文として最優先で表示する
**Objective:** 「My request for Codex」をユーザー入力の本文としてカード上部に配置し、アシスタントと同等の可読性でタイムラインに並べる。

#### Acceptance Criteria
1. WHEN ユーザーメッセージカードを表示するとき THEN 「My request for Codex」セクションが最上部に本文として配置され、他のセクションより先に視認できる。
2. WHEN ユーザーメッセージ本文を表示するとき THEN フォントサイズと行間はアシスタントメッセージと同一か同等のサイズでレンダリングされる。
3. WHEN ユーザーメッセージカードを表示するとき THEN 「IDE コンテキスト」や「My request for Codex」のメニュー/トグル UI は表示されない。

### Requirement 2: ユーザーメッセージ構造を本文＋任意オプションに簡素化する
**Objective:** ユーザーカードは「本文」と現状存在する任意のオプション項目のみで構成し、オプションはデフォルト未選択で表示する。

#### Acceptance Criteria
1. WHEN ユーザーメッセージを展開するとき THEN 「本文」セクションの下に現状のオプション項目（例: チェックボックス + オプションの値）が表示される。
2. WHEN 画面を初回表示するとき THEN すべてのオプション項目のチェック状態はデフォルトで OFF（未選択）になっている。
3. WHEN オプション項目が存在しない場合 THEN 「本文」単独でカードが成立し、不要な空のセクションは表示されない。

### Requirement 3: メインタイムラインと詳細情報のレイアウト比率を 7:3 に保つ
**Objective:** デスクトップ幅ではメインのメッセージ領域を右側詳細より優先し、約 70% : 30% の比率で表示する。

#### Acceptance Criteria
1. WHEN 画面幅がデスクトップ相当（例: `lg` ブレークポイント以上）でセッション詳細を表示するとき THEN 左カラム（メッセージタイムライン）が幅比率おおむね 70%（±5% 許容）となり、右カラム（詳細情報）が約 30% となる。
2. WHEN 画面幅がデスクトップ相当のとき THEN 右カラムが左カラムより広く表示されることはない。
3. WHEN 画面幅がモバイル～タブレット相当（`md` 以下）で表示するとき THEN メッセージ領域が横幅 100% で先に表示され、詳細情報はその下または後段に配置される。
